# Briaton Catalog

Учебный проект [Skillbox](https://skillbox.by/) по базовому JavaScript. Это статический каталог интернет-магазина светильников с фильтрацией и сортировкой товаров, пагинацией, корзиной, подсказками, слайдером и формой обратной связи. Проект работает на нативном JavaScript с модульной архитектурой.

## О проекте

Проект выполнен как финальная работа по JavaScript. При разработке использовалась готовая HTML-структура и стили.
Основная задача заключалась в реализации JS-функционала поверх подготовленной вёрстки согласно ТЗ.

## Ключевые особенности реализации

- модульная архитектура на нативном JS;
- реализация интерактивного поведения на основе готовой HTML-структуры и CSS;
- динамическая отрисовка товаров из `data/data.json`;
- фильтрация по типу и наличию товаров;
- сортировка по цене и популярности;
- пагинация каталога на клиенте;
- корзина с добавлением и удалением товаров;
- слайдер «Товары дня» на Swiper;
- всплывающие подсказки на Tippy.js;
- FAQ-аккордеон;
- валидация формы обратной связи на Just Validate.

## Функционал

- просмотр каталога товаров;

- фильтрация товаров по типу и наличию;
- сортировка по цене и популярности;

- добавление товаров в корзину и удаление из неё;

- просмотр подсказок по наличию товаров в городах;
- раскрытие ответов в блоке FAQ;
- отправка формы обратной связи.

## Быстрый старт

Проект не требует сборки. Для локального запуска используйте Live Server в VS Code.

### Локальный запуск

1. Откройте проект в VS Code.
2. Откройте `index.html`.
3. Запустите команду `Open with Live Server`.

После этого проект будет доступен в браузере по локальному адресу Live Server.

Примечание: прямой запуск `index.html` через браузер может нарушить загрузку данных из `data/data.json`.

## Структура проекта

```text
.
├── index.html          # Основная страница проекта
├── css/                # Готовые стили
├── data/
│   └── data.json       # Данные товаров
├── fonts/              # Локальные шрифты
├── images/             # Изображения и SVG-спрайт
├── js/
│   ├── api/            # Загрузка данных и отправка формы
│   ├── components/     # UI-компоненты и логика интерфейса
│   ├── helpers/        # Вспомогательные сущности для данных
│   ├── pages/          # Инициализация страниц
│   ├── vendor/         # Сторонние библиотеки
│   ├── config.js       # Конфигурация компонентов и сущностей
│   └── main.js         # Точка входа
└── scss/               # Исходники стилей
```

## Архитектура

- точка входа находится в `js/main.js`;
- логика страницы каталога собрана в `js/pages/CatalogPage.js`;
- переиспользуемые интерфейсные сущности вынесены в `js/components/`;
- данные товаров загружаются из `data/data.json` через `fetch`;
- форма обратной связи отправляется через отдельный API-слой в `js/api/MockApi.js`;
- проект не использует backend и работает как клиентское приложение на статическом хостинге.

## Соответствие заданию

В проекте реализованы основные требования ТЗ:

- открытие и закрытие бургер-меню каталога;
- открытие и закрытие меню выбора города со сменой активного значения;
- динамическая отрисовка карточек товаров на основе данных из `data/data.json`;
- отображение количества товаров рядом с фильтрами по категориям;
- фильтрация товаров по типу и наличию;
- сортировка товаров по трём параметрам: дешёвые, дорогие, популярные;
- добавление и удаление товаров в корзине с обновлением счётчика;
- функционал аккордеона в блоке «Ответы на частые вопросы»;
- слайдер в блоке «Товары дня» на базе Swiper.js;
- валидация формы обратной связи с помощью Just Validate;
- отправка формы на внешний тестовый endpoint с показом модального окна успеха или ошибки.

Дополнительно реализованы:

- пагинация каталога;
- всплывающие подсказки на карточках товаров с использованием Tippy.js.
